# MultiTableMax

Игровой тренажёр таблицы умножения для детей (~8 лет). JavaFX + Maven, JDK 25.

> Отображаемое название приложения — **«Тренажёр для Макса»**. `MultiTableMax` —
> технический идентификатор (пакет, artifactId, имя пакета jpackage); он оставлен
> латиницей, т.к. кириллица с пробелом ломает имена сборочных артефактов.

## Что внутри

- **Стартовый экран** — выбор режима и того, что учим, перед каждой игрой.
- **Три режима**:
  - *Обычный* — без ограничения по времени, играем сколько хотим;
  - *На время* — раунд на 60 секунд, в конце показывается счёт верных ответов;
  - *Босс-таблица* — нужно ответить без ошибок на все примеры области; даётся
    3 жизни, ошибка отнимает жизнь и пример вернётся позже. Решил все — победа.
- **Выбор области таблицы** — «Вся» (вперемешку 2–9) или произвольный набор
  множителей (например только ×3, или сразу ×3, ×4 и ×8). Выбранные таблицы
  подсвечиваются в панели прогресса.
- **Картинки боссов** — в режиме «босс» над примером показывается монстр.
  У каждой таблицы свой **отдельный персонаж**: слизень, циклоп, робот,
  привидение, чёртик, колючий, пушистик, дракончик и король (для «всей»/нескольких
  таблиц). Стиль настраивается на стартовом экране: *Emoji*, *Рисунок* (фигуры на
  Canvas) или *Картинки* (PNG из `resources/bosses`) — в трёх стилях это одни и те же
  персонажи.
- **Рекорды** — лучший результат «на время» и самое быстрое время победы над боссом;
  при новом рекорде на экране результата появляется «🏆 Новый рекорд!».
- **Экран статистики** — сводка ответов и точность, число освоенных примеров,
  самый быстрый ответ, рекорды, список самых трудных примеров и освоенность таблиц.
- **Сброс статистики** — кнопка на стартовом экране (с подтверждением).
- **Адаптивный тренажёр** — примеры показываются вперемешку; сложные (где были ошибки
  или которые ещё не освоены) выпадают чаще. В основе — система Лейтнера («коробки»).
- **Ввод с клавиатуры** — ответ цифрами, подтверждение по `Enter`.
- **Мгновенная обратная связь** — верно/неверно с показом правильного ответа.
- **Мотивация** — звёзды, серии (combo) с бонусами, звания (Новичок → Мастер),
  полоски освоения каждой таблицы 2–9.
- **Сохранение прогресса** — в `~/.multitablemax/progress.txt` (простой текст, без БД).

## Запуск при разработке

```bash
mvn clean javafx:run
```

## Автосборка под Windows (GitHub Actions)

При пуше в `main` (а также вручную через «Run workflow») запускается
`.github/workflows/build.yml`: на `windows-latest` ставится JDK 25 и WiX,
собирается jlink-образ и `jpackage` делает **`.msi`-установщик** с иконкой
(`packaging/app.ico` — король-босс). Готовый файл лежит в артефактах сборки
(`MultiTableMax-windows-installer`). Установщик включает свою JRE — у пользователя
Java не нужна. Иконка окна приложения берётся из `resources/bosses/boss-0.png`.

### Выпуск релиза (с прикреплённым .msi)

Релиз создаётся автоматически, когда пушишь **тег вида `vX.Y.Z`**. Тот же workflow
соберёт `.msi` и опубликует **GitHub Release** с этим файлом и авто-описанием.
Версия установщика берётся из тега (`v1.2.0` → `1.2.0`).

```bash
git tag v1.0.0          # тег ставится на текущий коммит
git push origin v1.0.0  # пуш тега запускает сборку и публикацию релиза
```

Через 5–10 минут готовый установщик появится на вкладке **Releases** репозитория —
его сможет скачать любой, без захода в Actions. Используй численные теги
(`v1.0.0`, `v1.2.3`); суффиксы вроде `v1.0-beta` jpackage не примет.

## Сборка нативного установщика вручную (.dmg / .exe / .msi / .deb)

Требуется JDK 25 (в нём есть `jpackage`). На Windows для `.exe`/`.msi` нужен
[WiX Toolset](https://wixtoolset.org/), на Linux для `.deb`/`.rpm` — `dpkg`/`rpmbuild`.

```bash
# 1. Собрать компактный runtime-образ с приложением (jlink)
mvn clean javafx:jlink

# 2. Завернуть образ в нативный установщик
mvn -Ppackage jpackage:jpackage
```

Готовый установщик появится в `target/dist/`.

> Кроссплатформенность: пакет нужно собирать на каждой целевой ОС отдельно
> (jpackage не умеет кросс-сборку). Образ `jlink` уже включает свою JRE —
> у конечного пользователя установленная Java не требуется.

## Структура

```
com.multitablemax
├── Launcher          точка входа для нативного пакета (не extends Application)
├── App               JavaFX-приложение, навигация между экранами
├── model
│   ├── Fact          пример a × b
│   ├── FactStats     статистика по примеру (коробка Лейтнера, серия, лучшее время)
│   └── Progress      весь прогресс: статистика + звёзды + уровень
├── engine
│   ├── GameMode      режим: PRACTICE / TIMED / BOSS (+ параметры раундов)
│   ├── QuizEngine    ядро: область таблицы, взвешенный выбор, начисление звёзд
│   └── BossRun       раунд «босс»: очередь примеров, жизни, победа/поражение
├── store
│   └── ProgressStore загрузка/сохранение в текстовый файл
└── ui
    ├── HomeView      стартовый экран: выбор режима, области таблицы, стиля боссов
    ├── StatsView     экран статистики
    ├── BossArt       стиль картинок боссов: EMOJI / CANVAS / IMAGE
    ├── BossArtist    отрисовка босса (emoji / Canvas / PNG из ресурсов)
    └── TrainerView   экран тренажёра (обычный / на время / босс)
```

Ядро (`model`/`engine`/`store`) не зависит от JavaFX — игровые режимы можно
добавлять как новые экраны поверх него, не трогая логику.

## Картинки боссов (PNG)

Файлы `src/main/resources/bosses/boss-*.png` — оригинальная графика, сгенерированная
для проекта (Java2D), и распространяется как общественное достояние (public domain):
использовать/менять можно свободно, без указания авторства. Заменить на свои или на
CC0-набор (например, Kenney.nl) можно, просто положив PNG с теми же именами
(`boss-0.png` — общий босс, `boss-2.png` … `boss-9.png` — по таблицам).
